🐛 🔴 重大Bug发现与修复：Alpha派 cron 扫描 `-type` 参数缺失
📌根因

SOP v3.72 文档和本地 task 文件中的 alphapai_client.py recall 命令示例全部漏了 --type 参数前缀：

`bash

# ❌ 错误（cron 实际执行，exit code 2）

python3 scripts/alphapai_client.py recall --query "特斯拉" comment,roadShow --start 2026-05-14 --end 2026-05-15

# ✅ 正确

python3 scripts/alphapai_client.py recall --query "特斯拉" --type comment,roadShow --start 2026-05-14 --end 2026-05-15

`

comment,roadShow 被当作位置参数直接报错退出，cron 的早间（二）9词仅"玻璃基板"命中（可能恰好作为位置参数被忽略后走了默认行为），其余8词全空。

📌影响范围
  • 2026-05-15 早间（二）cron 报告 10 条事件（仅玻璃基板），实际重新扫描后 159 条原始→87 条去重→31 条入选
  • 可能影响更早的扫描（但未被发现，因为总有部分词命中）
🐛修复（4处） Bug Fix

1. SOP 飞书文档 4.8 节深挖代码示例：三处 comment,roadShow → --type comment,roadShow，一处 roadShow_ir → --type roadShow_ir，代码块首行加醒目警告注释

2. SOP 第九部分检查命令：简化写法 → 完整命令含 --type

3. SOP 第八部分核心铁律新增第 18 条：--type 参数铁律

4. SOP 版本 v3.72 → v3.73

5. 本地 tasks/ai-chain-daily-scan.md 和 tasks/comein-brm-ai-chain-daily-scan.md 中对应的代码示例同步修复

⚠️教训 教训
  • 命令示例中的每个参数都要明确标注参数名，不能依赖位置参数
  • cron 输出"0 条"时要怀疑是命令格式问题而非数据稀疏
  • 应该在核心铁律速查区加入关键约束，cron agent 每次执行前扫一遍
