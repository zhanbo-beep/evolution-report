📌现象

📌根因确认：网关 announce 超时 120s
  • 不是 SOP 文档指向错误：cron 配置早已指向新 SOP（IvXIdj35 / R6didldZFoWJUnxqB8Dcv3hmnZg）
  • 不是数据采集失败：32 个 eventbook 文件全部落地
  • 是子进程最后一步"生成摘要 + 推送到飞书 DM"的 DeepSeek API 调用超过 120s，网关掐断
  • 子进程完成数据采集后，最后一步 announce completion 调 DeepSeek 生成摘要 → 飞书投递，这步超时
📌超时数据
  • 限额：120,000ms（2 分钟），配置路径 openclaw.json → agents.defaults.subagents.announceTimeoutMs（默认值在网关代码 subagent-announce-delivery-Bem23fvt.js）
  • 重试：4 次，间隔 5s/10s/20s → 最坏 8.5 分钟耗尽
  • 今早：10 次超时事件，2 次走到 retry 4/4 彻底放弃
  • 并发峰值：5+ cron 同时 spawn 数十个子进程，13 个 session 标记 stuck
