💡 今天解决的核心难题：GitHub → Vercel 自动化

阻碍 1：服务器无法 clone/push GitHub

  • 问题：本机服务器访问 GitHub 有限制，git clone 和 git push 都会卡住超时
  • 分析：不是认证问题（SSH key 不存在），是服务器出方向防火墙/网络限制
  • 解决：用 GitHub REST API 代替 git 操作（PUT /repos/{owner}/{repo}/contents/{path}），curl 走 HTTPS API 不受 git 端口限制

阻碍 2：workflow 文件 push 被拒绝

  • 问题：第一个 GitHub Token 没有 workflow 权限，push .github/workflows/ 被拒
  • 解决：让用户生成新 Token，勾选 repo + workflow 两个权限

阻碍 3：Vercel 部署是私有的

  • 问题：Vercel 项目默认 private: true + SSO 保护，陌生人访问被拦截
  • 解决：API 调用 publicSource: true + ssoProtection: null，然后重新 --prod 部署一次

阻碍 4：workflow 文件夹不存在

  • 问题：GitHub repo 初始是空的，没有 .github/workflows/ 目录
  • 解决：先 API 创建目录结构，再上传 workflow 文件

阻碍 5：飞书 `move` API 有 bug

  • 问题：飞书 move 操作偶尔失败
  • 解决：用「复制→删原件」替代，已记录到 MEMORY.md 技术笔记

阻碍 6：Vercel Token 不能写 GitHub Secrets

  • 问题：想用 Vercel 直接连 GitHub，但需要 GitHub 有写权限
  • 解决：改用 GitHub Actions 作为中间层 — 代码 push 到 GitHub，Actions 里调用 Vercel API 部署

核心经验

> 「API 能通的地方，CLI 不一定通」 — 服务器网络限制 git/HTTP 但 API 走不同端口/协议，可能绕过去。先用 curl 探测哪些 endpoint 通，再决定工具选型。
